# CreativeReporter 구조와 매커니즘

퍼포먼스 소재 성과분석 대시보드의 전체 설계 문서. 이 문서만 읽으면 시스템이 어떻게 동작하는지, 왜 그렇게 설계됐는지 알 수 있다.

## 1. 한눈에 보기

```
[Meta/Moloco/Google CSV]                [Google Ads 보고서들]
        │ 드래그&드롭                        │ (광고그룹 보고서·이름 매핑)
        ▼                                   ▼
┌──────────────────────────────────────────────────────┐
│  index.html (단일 파일, 바닐라 JS/CSS/SVG, 의존성 0)   │
│  파싱 → 매체 판별 → 필터 → 뷰 누적 → 채점 → 렌더        │
└──────────────┬───────────────────────────────────────┘
               │ RPC (비밀번호 게이트)
               ▼
┌──────────────────────────────────────────────────────┐
│  Supabase (Postgres)                                  │
│  weekly_creative_stats  ← 주간 성과 원본 (JSONB)        │
│  cr_kv                  ← gmap·ginfo·ocr·ytt·nameovr·svcovr│
│  cr_users               ← 사용자 계정·승인 상태·권한       │
│  cr_user_marks          ← 즐겨찾기·내 소재 (계정 단위)     │
└──────────────────────────────────────────────────────┘

배포: GitHub Pages (main 머지 → Actions 자동 배포, 실패 시 백오프 재시도 3회)
관리: admin.html — 가입 신청 승인/거절 (관리자 비밀번호 게이트, 저장소와 별도 URL)
```

- **프론트엔드**: `index.html` 하나. 프레임워크·번들러·외부 라이브러리 없음. 모든 계산(집계·채점·차트)은 브라우저에서 수행.
- **백엔드**: Supabase Postgres. 클라이언트는 REST RPC만 호출. 테이블 직접 접근은 RLS로 전면 차단.
- **호스팅**: GitHub Pages. `main`에 머지하면 `.github/workflows/pages.yml`이 배포(일시 거부 대비 90초·5분 백오프 재시도).

## 2. 데이터 모델

### weekly_creative_stats — 주간 성과 원본
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | bigint identity | PK |
| channel | text | `Meta` `Moloco` `Google` (그 외·'기타'는 서버가 거부) |
| week_start | date | 주 시작일(월요일) |
| ad_name | text | 소재 식별자 (구글은 소재 URL) |
| payload | jsonb | 그 주의 원본 수치 전부 |
| uploaded_by | text | 업로더 서명 (`/hansy` 입력 시 `hansy`, 기본 `unknown`) |
| uploaded_at | timestamptz | 자동 기록 |
| cost_krw·impressions·clicks·installs·active_users | numeric **generated** | payload에서 자동 파생(콘솔 조회용, 쓰기 불필요) |

- **UNIQUE(channel, week_start, ad_name)** — 같은 매체·주차·소재는 한 번만. 재업로드 중복은 DB 레벨에서 무시된다.
- **payload가 JSONB인 이유**: 매체마다 지표 스키마가 다르고(메타 ~14종, 몰로코 ~20종, 구글 ~15종) 새 지표가 계속 생긴다. 컬럼 분리 시 대부분 NULL인 25+ 컬럼과 지표 추가 때마다 DDL 변경이 필요해진다. 사이트는 행 전체를 통째로 읽어 브라우저에서 집계하므로 컬럼 단위 쿼리 이점도 없다. 식별·필터 키(channel/week_start/ad_name/업로더)는 정식 컬럼으로 분리되어 있고, 자주 보는 수치는 generated column으로 노출한다 — "쿼리는 컬럼, 내용물은 JSONB" 하이브리드.

payload 필드(합집합): `비용, 노출 수, 클릭 수, 어트리뷰션 수, 활성 유저 수, 신규+재활성 유저 수, _m_installs, _vcrNum, _vcrDen(완주율 분자/분모), _v25~_v100(동영상 25/50/75/95/100% 구간별 재생수 — 있으면), _g_interactions, _g_trueViews, _g_conversions, _g_allConv, _g_convValue` + 문자열 `_m_url, _m_ctype, _m_res, _m_group, _g_assetStatus, _g_assetType, _g_status, _g_level, _g_url, _g_assetId, _g_adGroupId, _g_campaignId`

### cr_kv — 보조 저장소 (k, v jsonb, updated_at, updated_by)
| 키 | 내용 | 채워지는 경로 |
|---|---|---|
| `gmap` | `{adgroup:{id:이름}, campaign:{id:이름}, asset:{항목ID:공식이름}}` | 이름 매핑 CSV·광고그룹 보고서 드롭, 이미지 매칭 배치 |
| `ginfo` | `{광고그룹ID:{nm,cpNm,st,op,rsn,tcpa,cost,…,v25~v100,from,to}}` | 광고그룹 보고서 드롭 |
| `ocr` | `{항목ID:이미지에서 추출한 광고 카피}` | PaddleOCR 오프라인 배치 |
| `ytt` | `{유튜브ID:영상 제목}` | 사이트가 로드 시 자동 조회(noembed) 후 저장 |

### cr_users — 사용자 계정과 승인 워크플로
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | bigint identity | PK |
| daangn_name | text | 사용자명(영문 시작 2~24자 영문·숫자). 앞으로 수정·제작·즐겨찾기 등 행위의 서명(수정자) 값 |
| name_lower | text **generated** unique | `lower(daangn_name)` — 대소문자 무관 유일성(사칭 방지) |
| pw_hash | text **unique** | 비밀번호 sha256 해시. 로그인·게이트가 비번만으로 사용자를 식별하므로 전역 유일 |
| pw_enc | bytea | 비밀번호 **복호화 가능 암호문**(pgcrypto pgp_sym, 키는 서버 함수 본문에만). 관리자가 비번을 열람·관리하기 위한 용도 — 관리자 게이트 RPC 안에서만 복호화 |
| status | text | `신청함`(가입 신청, 로그인 불가) · `승인됨`(로그인·데이터 접근 가능) · `거절됨`(로그인 불가, 기록 보존·재승인 가능) |
| **role** | text | `viewer`(보기 전용, 기본값) · `editor`(저장·수정 가능) · `admin`(에디터 상위 — 소재 이름 변경까지). 관리자가 admin.html에서 지정. 현재 admin은 hansy만 |
| created_at·reviewed_at·reviewed_by | | 신청 시각 · 승인/거절/권한 처리 시각·처리자 |

- **비밀번호만으로 로그인**: 입력 비번의 sha256 해시로 사용자를 찾아 이름·상태·권한을 반환(`cr_login`). 그래서 pw_hash를 전역 유일로 강제(가입 시 중복 비번 거부).
- **레거시 `0715`는 이름 없는 뷰어 하드코딩 로그인** — cr_login이 `{viewer:true}`를 반환(이름 없음), 데이터는 읽되 저장·수정은 불가. cr_users 행이 아니므로 가입 시 예약 비번 `0715`는 거부.
- 권한 모델: **뷰어**(0715 또는 role=viewer 사용자) = 읽기 전용 · **에디터**(승인됨 + role=editor) = 저장·수정 가능 · **관리자**(승인됨 + role=admin) = 에디터 권한 + 소재 이름 수동 변경(`/namechange`). 관리자는 에디터 권한을 포함(canEdit=true). 소유자는 계정을 만들어 admin.html에서 자신을 에디터/관리자로 승인. 관리자용 로그인 계정 `admin`(editor) 시드. **관리자 권한(admin role)은 admin.html 마스터 게이트(별도 시크릿)와 다른 개념** — role=admin은 앱 내 최상위 사용자 등급, `_cr_check_admin`은 계정 관리 콘솔용 마스터 비번.

### cr_user_marks — 즐겨찾기 / 내 소재 (개별 계정 단위)
| 컬럼 | 설명 |
|---|---|
| user_name | 마크 소유자 daangn_name |
| channel · ad_name | 대상 소재 (매체, 소재 식별자) |
| kind | `fav`(즐겨찾기, 비공개) · `mine`(내 소재, 등록자 공개) |
- UNIQUE(user_name, channel, ad_name, kind). RLS deny-all — RPC로만.
- **뷰어도 마크 가능**(계정만 있으면). 0715는 계정이 없어 불가. 쓰기 게이트가 아니라 `_cr_uname(pw)`로 비번→사용자 도출(0715/미존재는 null→거부).
- `cr_marks_load`는 **내 소재는 전원 공개(등록자 이름 포함)**, **즐겨찾기는 본인 것만** 반환(프라이버시).

## 3. 보안 모델

- 세 테이블 모두 **RLS deny-all**: anon 키로는 SELECT/INSERT 자체가 불가능.
- 모든 데이터 접근은 **SECURITY DEFINER RPC**를 통해서만 하고, 각 RPC 첫 줄에서 `_cr_check_pw(pw)`가 비밀번호를 검사(불일치 시 예외).
- **읽기 게이트 `_cr_check_pw`**: 입력 비번이 레거시 `0715` 이거나 **승인됨 사용자의 비번**이면 통과(읽기). 신청함·거절됨은 차단.
- **쓰기 게이트 `_cr_check_editor`**: `cr_save`·`cr_kv_merge`(저장·이름매핑·그룹정보·유튜브제목)는 **승인됨 + role=editor** 만 통과. 0715·뷰어는 거부(프론트도 `canEdit`로 미리 막아 드롭은 분석만 되고 저장은 건너뜀).
- 클라이언트의 publishable key는 공개되어도 되는 값(설계상 공개키). 실질 게이트는 비밀번호.
- 비밀번호는 브라우저 sessionStorage(탭 단위)에만 저장(`crpw`) — 새로고침 유지, 탭 종료 시 소멸. 로그인한 daangn name은 `crname`(탭)·`cr_uploader`(localStorage, 기기)에 저장돼 업로드·수정 서명에 쓰인다.
- **관리자 게이트**는 사용자 비번과 완전히 분리된 별도 시크릿(`_cr_check_admin`). admin.html에서 런타임 입력만 하고 **리포지토리엔 저장하지 않는다**(GitHub Pages 공개 파일). 관리 RPC(`cr_admin_*`)만 이 시크릿으로 게이트.
- 회원가입·로그인은 anon 키로 공개 호출 가능(신청은 누구나, 로그인은 승인·비번 필요) — 기존 보안 태세와 동일.

### 로그인·가입 흐름
1. **가입**: 랜딩 '회원가입' → daangn name(영문 시작 2~24자)·비번·비번확인 → `cr_register`(신청함 적재). 이름/비번 중복·형식 오류는 서버가 거부(`name_taken`/`pw_taken`/`name_invalid`/`pw_weak`). 성공 시 "관리자 확인 후 가입이 승인됩니다".
2. **관리자 처리**: admin.html에서 신청함 → 승인됨/거절됨. 거절은 삭제가 아니라 상태 저장이라 언제든 재승인 가능.
3. **로그인**: 비밀번호만 입력 → `cr_login`이 `{found,name,status,role,viewer}` 반환. 인사(탭당 1회) — 0715 뷰어="뷰어 권한으로 로그인했어요 👀"(이름 없음, 환영 문구 없음) · 승인된 사용자(에디터·뷰어 공통)="NAME님 환영해요! 🥕"(환영 문구만) · 신청함="아직 관리자 승인 대기 중이에요" · 거절됨="가입이 거절된 계정이에요" · 미존재=흔들기. (모든 토스트는 당근 보이스 해요체)
- **비밀번호 규칙**(`_cr_pwok`): **4자 이상**(복잡도 요구 없음). **같은 비밀번호로 다른 계정 생성 금지**(pw_hash 전역 유일 → register·admin_create·setpw 모두 `pw_taken`로 차단 → "다른 걸로 정해 주세요"). 예약비번 `0715`만 별도 거부.
- **새 가입 신청 알림**(관리자 전용): admin.html이 열려 있는 동안 15초 폴링해 새 신청 시 토스트+브라우저 알림, **거절은 확인창 없이 즉시**. (메인 화면에는 알림 없음 — 관리자만 봐야 하므로)
- **관리자 페이지**: 비밀번호 열은 평소 ●●●, **마우스 호버 시 표시**(클릭 복사). daangn name **✎로 수정**(cr_admin_set p_name). 처리 열은 날짜만(처리자 'admin' 미표기). ＋계정 추가·비번변경.
- **즐겨찾기 / 내 소재**(로그인 계정, 뷰어 포함 · 0715 제외): 소재 이름 우측 별 버튼 2종 — ★ 즐겨찾기(켜면 노랑) · 체크 인장 내 소재(켜면 하늘색). **표시 시(on) 별은 호버 없이 상시 노출**, 미표시는 호버 시에만 회색. **즐겨찾기는 별만**(칩·글자 없음) · **내 소재는 남이 등록한 것만 등록자 이름 칩**(본인 것은 켜진 하늘색 별로 충분하므로 칩 생략). 별은 **전체 목록·그룹 행·상세의 "같은 그룹 내 비교" 목록** 모두에 표시. 좌측 nav "보기": 전체 / 내 소재 / 즐겨찾기 필터.
- **그룹 마크(파생·일괄)**: 그룹뷰에서 그룹 별을 누르면 멤버 전원을 일괄 토글(`cr_mark_set_many`). 그룹 상태는 **저장하지 않고 멤버에서 파생** — 멤버 전원이 표시면 그룹도 표시로 보이고, 멤버 하나를 개별 해제하면 그룹 표시도 자동 해제. 표·비교 목록은 전체 재렌더 없이 제자리 갱신(`refreshMarksUI`)해 열린 상세가 닫히지 않는다.
- **소재 편집(관리자, 연필 팝오버)**: 별(즐겨찾기·내 소재) 오른쪽 **연필 버튼**(관리자·개별 소재만, 호버 노출) → 그 소재의 **이름·서비스 분류·제작자(내 소재 등록자)** 를 한 팝오버에서 수정. 저장 시 이름=`cr_names_merge`(kv `nameovr`), 서비스=`cr_svc_merge`(kv `svcovr`, ''=자동 복원), 제작자 추가/삭제=`cr_admin_mark_set`(특정인 mine 마크 대신 등록/해제). 표시 이름 우선순위 최상위, 서비스 오버라이드는 `svcOf`·드롭다운·서비스 칩(주황)에 즉시 반영.
- **소재 이름 일괄 변경(관리자, `/namechange`)**: 검색창에 `/namechange`+엔터 → 모든 소재명 contenteditable, 다시 `/namechange`로 저장(`cr_names_merge`). 저장 전 이탈은 `beforeunload`로 붙잡음. (연필 팝오버가 개별 편집의 주 경로, `/namechange`는 일괄용)
- **서비스·제작자 복수선택 드롭다운 필터**: 툴바에 `전체 서비스`·`전체 제작자` 2종(체크박스 복수선택, 팝업 유지). 서비스=이름·태그·그룹명 키워드 6버킷(중고거래·브랜딩(브랜딩·보검)·부동산·알바·비즈니스(로컬애즈·비즈)·페이(픽업·래플))+기타, 관리자 오버라이드(`svcOf`) 우선. 제작자=내 소재 등록자 목록(내 소재 마크가 있을 때만 노출). 개수는 현재 화면(개별/그룹) 기준.
- **OCR 마크**: 소재명 옆이 아니라 **상세보기 미리보기의 애셋 URL 왼쪽**에 표시. **Live**는 초록 점+글자를 감싼 **칩** 형태.

### RPC 목록

## 4. 파일 인식과 저장 파이프라인

드롭된 파일은 이 순서로 분류된다:

1. **구글 확장 소재 연결 보고서** — 헤더에 `애셋 상태`+`확장 소재`. `주` 컬럼(주간) 파싱, 지표 매핑(비용/노출수/클릭수/설치/인앱 액션/전환 등 → 내부 키).
2. **몰로코 크리에이티브 리포트** — `creative_preview` 또는 `ad_name`+`creative_type`. 미리보기 CDN 링크·타입·해상도·그룹 추출. **네이티브 영상 오등록 보정**: 리포트가 완주율을 제공하는데 `creative_type`이 VIDEO인 소재의 첨부가 이미지 파일이고 완주율이 전 기간 비어 있으면 실제 노출물을 이미지로 보고 이미지로 재분류(칩 `이미지*`·유형 '이미지(영상 등록)'로 표기). 완주율이 찍혀 있으면 미리보기만 썸네일인 진짜 영상이므로 유지, 완주율 컬럼 자체가 없으면 판단 보류.
3. **일반 성과 CSV(메타 등)** — `date`+`ad_name`(또는 세트/그룹 레벨 컬럼). `thumbnail_url` 류 컬럼이 있으면 미리보기 자동 연동. `동영상 … 25/50/75/95/100%` 재생수 컬럼이 있으면 자동 인식해 `_v25~_v100`으로 누적 — 상세 '영상 시청' 탭의 구간별 유지 곡선 데이터(율·평균·시간·비용류 헤더는 제외, 횟수만).
4. **광고그룹 보고서** — 헤더에 `광고그룹 상태`+`광고그룹 ID`. 그룹 메타+누적 성과+영상 재생 퍼널을 `ginfo`로, 이름을 `gmap`으로.
5. **이름 매핑 CSV** — `광고그룹 ID`+`광고그룹` / `캠페인 ID`+`캠페인` / `항목 ID`+`애셋` 쌍. 제목 줄이 앞에 있는 UI 내보내기도 지원.

**매체 판별 우선순위**: ① 리포트 형식 자동 감지 → ② 미리보기 URL 도메인(adsmoloco→Moloco, googlesyndication/youtube→Google, fbcdn→Meta) → ③ 드롭 위치 → ④ 파일명 접두(meta_/moloco_/google_).

**저장 규칙**:
- 주간 리포트만 저장소 누적. 월간·일간·기간미상·매체미상은 **임시 분석 모드**(배너 표시, 저장 안 함, ☁️ 클릭으로 복귀).
- (주차,소재) 병합 그레인으로 합산 광고비 ₩1,500 미만이면 분석·저장 모두 제외.
- 뷰 누적도 DB와 같은 규칙: 이미 있는 (주차,소재)는 유지, 새 것만 추가 — 재드롭해도 이중 집계 없음.
- **확인창 규칙**: 광고그룹 보고서·이름 매핑처럼 기존 저장 정보를 *덮어쓸 수 있는* 파일은, 기존 값과 달라지는 항목이 있을 때만 예시와 함께 확인창(확인=ID 기준 갱신 / 취소=미반영). 신규 항목뿐이면 확인 없이 바로 반영.

## 5. 표시 이름 시스템 (ID/URL → 사람이 읽는 이름)

`displayNameOf(entity)` 우선순위:

1. **그룹 행** → `groupPrettyName`: 이름을 `_`로 분절해 `중고거래·shortform·2601` 형태로. `maugrowth`는 상수 접두라 생략, 그 앞의 다른 접두(`Google UA` 등)만 표시. 서비스 토큰은 한글 사전(VERTICALS: fleamarket→중고거래, realty→부동산, jobs→알바 …)으로 변환.
2. **공식 애셋 이름** — `gmap.asset[항목ID]` (이름 매핑 CSV 또는 이미지 매칭 배치가 채움)
3. **OCR 이름** — `ocr[항목ID]` (이미지 속 광고 카피, `OCR` 마크 부착)
4. **유튜브 제목** — `ytt[영상ID]` (로드 직후 전 소재 자동 조회·kv 공유)
5. 원문 (data-name·저장·병합 키는 항상 원문 유지, hover 시 원문 툴팁)

### 오프라인 이름 보강 배치 (필요 시 재실행)
- **PaddleOCR(korean)**: 구글 이미지 애셋 200장 → 다운로드 1회 → 1100px 그레이 리사이즈 → OCR. 핵심 설정: `OMP_THREAD_LIMIT=1`(스레드 폭발 방지), 숫자 포함 라인은 신뢰도 0.55까지 수용, **박스 좌표 기하 정렬**(위→아래, 같은 줄은 좌→우)로 문장 순서 보정, 후처리(로고 오인 접두 `6당근`/`90 당근` 제거, 선두 단독 '당근' 제거, 말미 아이콘 오인 제거). 200장 ≈ 5분.
- **dHash 이미지 매칭**: 몰로코 IMAGE 소재(이름 있음)와 구글 simgad 이미지의 지각 해시(9×8 dHash, 해밍 ≤6 + 종횡비 ±12%)로 동일 이미지를 찾아 몰로코의 실명을 구글 애셋에 부여 → `gmap.asset`.
- 취소: 해당 kv 키 삭제 한 번이면 전체 원복.

## 6. 채점 시스템

```
Score = 100 × (0.75·E + 0.20·R + 0.05·C)
  E = Σ wᵢ·g′ᵢ   g′ = 0.5 + (g−0.5)·n/(n+n₀)      ← 표본 축소(shrinkage)
      g = 광고비 가중 백분위 (buildWDist/wPctRank)   n₀: 전환류 50 · 클릭 100 · 노출 2,000
      CTR·CPM·CPC·완주율·상호작용률류는 같은 포맷(영상/이미지) 코호트(≥8개) 안에서만 상대평가
  R = pctRank(노출 수)                               ← 도달 규모 백분위
  C = min(1, ln(1+전환)/ln(1+30))                    ← 표본 신뢰 감쇠
  표시 점수 = 원점수(위 결과)를 비교군 내 위치로 선형 재배치 후 소수점 1자리
      앵커: 원점수 5백분위→12점 · 95백분위→92점, 양 끝은 점근 압축(최고 99 · 최저 2, 순위 완전 보존)
      비교군 8개 미만이면 분위 앵커가 불안정해 원점수 그대로 사용
```

- 매체별 지표 가중치(SCORING): **ad(메타·몰로코)** eCPI .26 / 활성 eCPA .20 / CTR .16 / 신규 eCPA .12 / CPM .10 / 완주율 .08(영상만) / CPC .08 · **google** ROAS .35 / 전환당 비용 .20 / 전환율 .15 / CTR .15 / CPM .15 · **adset** eCPA 중심 · **cg** CTR 중심.
- 지표 결측 시 가중치 자동 재정규화. 분모가 존재하면 성과 0도 유효값(예: 전환가치 0원의 ROAS=0 — null로 두면 평가 회피 발생).
- **eligibility**: 광고비 15퍼센타일 이상 + 핵심 지표(가중 .15↑) 보유. 미달이면 점수 없음(–).
- **학습중 배지**: 집행 2주 미만 또는 전환 30건 미만.
- 등급: 상대 5분위(매우 좋음~매우 나쁨) + '독보적'(압도적 격차, 광고비 중앙값 이상만).
- 그룹 행도 동일 스펙으로 별도 채점(scoreGroupAgg).

## 7. UI 구성

- **사이드바(다크)**: 매체(소재 수 많은 순, 기간 반영 카운트) · 기간(최근 1달/3달/전체/직접 설정 — 최근 N 주·달 입력식) · 요약 KPI(총광고비·설치·eCPI·활성·eCPA, 주간 증감) · ☁️ 저장소 줄(소재/주간 기록/주 수 + 업로더 툴팁) · 용어/CSV 버튼. 좁은 화면: 매체·기간 2분할, 나머지는 '요약' 토글(기본 접힘).
- **세그먼트 탭**: 분석 / 전체 목록 / AB테스트(인식 그룹 0개면 탭 숨김). 본문은 카드 없이 흰 배경 풀폭, 푸터는 캔버스색+간격으로 구분.
- **분석**: 요약 타일(포맷/사이즈 우위, 주간 흐름) · 효율 지도(2축 산점도 — '보기' 세그먼트로 개별 소재 또는 포맷/사이즈/형태/지면/서비스별 집계(상하 10% 트림) 전환, 가로/세로 축도 세그먼트, 범위 슬라이더. 구 '유형별 비교' 그래프는 지도에 흡수) · 주차별 추세(선택 기간 전체, 라벨 자동 솎기).
- **전체 목록**: 컬럼 계층(대표: 점수·노출·eCPI·eCPA·CTR·완주율·ROAS, 헤더 +로 세부 펼침) · 기본 Live 필터(스코프 최신 집행 주 광고비>0) + 무점수(–) 소재 기본 숨김('비활성·점수없음도 보기'로 표시) · GroupView(광고그룹 집계, Video/Image와 상호 배타) · 이름 아래 칩 클릭=필터(서비스·포맷·사이즈·그룹·날짜) · 60행 페이지. 점수 칸은 10단계 색(십의 자리마다 다른 색, 파랑→청록→초록→올리브→무채→황토→주황→빨강).
- **이름 옆 뱃지**: `Live`(초록 점 — 최신 집행 주에 광고비가 나간 소재/그룹, 전 매체 공통) · `경고`(호박색 — 구글 광고그룹 보고서상 정책 제한, 사유는 툴팁) · `학습중`(집행 2주 미만 또는 전환 30건 미만). 일시중지는 뱃지 없음 — Live 뱃지 부재로 드러나고, 그룹 상세 메타에만 '일시중지' 텍스트 표기.
- **상세(행 클릭)**: 한 줄 3열 [고정 미디어 프레임 | 그래프 탭 | 같은 그룹/시리즈 비교] — 3분할 구조 고정. 가운데 열은 세그먼트 탭 3종, 있는 데이터의 탭만 노출(1개면 탭 줄 숨김)하고 선택 탭은 소재 이동(←/→·비교 클릭) 간 유지: ① **주차별 추이**(막대 광고비+지표선, 매 막대 아래 MM/DD와 **그 주 점수** — 그 주 데이터만으로 재채점한 상대 점수(weeklyScoresOf, 스코프 캐시), 티어 색·부적격 주는 –) ② **영상 시청**(영상 소재만) — 구간별 시청 유지 곡선(노출=100 기준, 최대 이탈 구간 음영·−%p 라벨. 데이터 우선순위: 소재 자체 `_v25~_v100`÷노출 → 구글 광고그룹 보고서 누적 퍼널(그룹 단위임을 캡션 명시) → 구간 데이터 없는 매체는 완주율 큰 값+코호트 중앙값 대비) + 완주율/TrueView 조회율 주차별 추이(영상 중앙값 점선) ③ **포지셔닝** — 두 축 모두 '계정 내 위치(광고비 가중 백분위, 방향 보정 0~100)'인 사분면 산점도: 어떤 지표를 골라도 오른쪽·위=좋음. 방향 표시는 미니멀하게 — 중앙값 십자선에 나쁨(빨강)→좋음(초록) 그라데이션 + 선 끝 +/− 글리프만(사분면 틴트·코너 문구는 점과 겹쳐 제거). 축 지표는 가로/세로 토글로 교체(기본 가로 CTR류·세로 효율류, 선택은 소재 이동 간 유지). 회색=타 소재·파랑=같은 그룹·주황=이 소재, 점 크기 광고비, 점 클릭=해당 소재로 이동. 원값 스케일 분포는 분석 탭 효율 지도가 담당(역할 분담). + 지표별 계정 내 위치 막대. 점수·진단 블록 없음(점수는 행에서, 산정 방식은 푸터에서). 비교 목록은 한 줄 말줄임+공통 접두 '…' 접기(풀네임 툴팁). 프레임은 비율별 3종(세로 232×396/정방 280×280/가로 400×250), 모바일(≤860)은 미디어→비교→그래프 순 1열 스택+프레임 폭 캡. 그룹 상세는 미디어 자리에 보고서 추가 정보(동영상 재생 퍼널은 '영상 시청' 탭이 담당). 키보드 ←/→·Esc, 딥링크 해시 기록.
- **이미지 로딩 복구**: 광고차단 DNS가 광고 CDN(googlesyndication·adsmoloco)을 거부하는 환경 대응. 실패 즉시(대기 0ms) wsrv.nl 프록시로 전환 → (1.2s) 원본+캐시우회 → (1.2s) 프록시+캐시우회 → 실패 표시. 프록시로 성공하면 `crImgProxy`를 localStorage에 학습해 그 기기는 재방문 때도 처음부터 프록시로 렌더(우회 재시도 자체가 없어짐). 모든 이미지 태그는 `data-osrc`에 원본 URL 보존. 호버 툴팁의 유튜브 썸네일은 표에서 이미 캐시된 저해상도(default.jpg)를 즉시 띄우고 고해상도(hqdefault)가 준비되면 교체.
- **AB테스트**: 이름 접미 변형 자동 인식(`_A`/`_B`, v1·v2, 제로패딩 01~05, ep·편·화 회차, 문자 직결 숫자 1~5 — 오인 방지 규칙 다수). 목록(이름+🥇🥈🥉)→펼침 비교표(유의미 차이만 색상, 5% 스프레드 기준)+대표 미디어.
