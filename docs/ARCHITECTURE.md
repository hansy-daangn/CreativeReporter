# CreativeReporter 구조와 매커니즘

퍼포먼스 소재 성과분석 대시보드의 전체 설계 문서. 이 문서만 읽으면 시스템이 어떻게 동작하는지, 왜 그렇게 설계됐는지 알 수 있다.

## 1. 한눈에 보기

```
[Meta/Moloco/Google CSV]                [Google Ads 보고서들]
        │ 드래그&드롭                        │ (광고그룹 보고서·이름 매핑)
        ▼                                   ▼
┌──────────────────────────────────────────────────────┐
│  index.html (단일 파일, 바닐라 JS/CSS/SVG, 의존성 0)   │
│  파싱 → 매체 판별 → 필터 → 뷰 누적 → 채점 → 렌더        │
└──────────────┬───────────────────────────────────────┘
               │ RPC (비밀번호 게이트)
               ▼
┌──────────────────────────────────────────────────────┐
│  Supabase (Postgres)                                  │
│  sr_weekly_creative_stats  ← 주간 성과 원본 (JSONB)        │
│  sr_kv                  ← gmap·ginfo·ocr·ytt·nameovr·svcovr│
│  sr_users               ← 사용자 계정·승인 상태·권한       │
│  sr_user_marks          ← 즐겨찾기·내 소재 (계정 단위)     │
└──────────────────────────────────────────────────────┘

배포: GitHub Pages (main 머지 → Actions 자동 배포, 실패 시 백오프 재시도 3회)
관리: admin.html — 가입 신청 승인/거절 (관리자 비밀번호 게이트, 저장소와 별도 URL)
```

- **프론트엔드**: `index.html` 하나. 프레임워크·번들러·외부 라이브러리 없음. 모든 계산(집계·채점·차트)은 브라우저에서 수행.
- **백엔드**: Supabase Postgres. 클라이언트는 REST RPC만 호출. 테이블 직접 접근은 RLS로 전면 차단.
- **호스팅**: GitHub Pages. `main`에 머지하면 `.github/workflows/pages.yml`이 배포(일시 거부 대비 90초·5분 백오프 재시도).

## 2. 데이터 모델

### sr_weekly_creative_stats — 주간 성과 원본
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | bigint identity | PK |
| channel | text | `Meta` `Moloco` `Google` (그 외·'기타'는 서버가 거부) |
| week_start | date | 주 시작일(월요일) |
| ad_name | text | 소재 식별자 (구글은 소재 URL) |
| payload | jsonb | 그 주의 원본 수치 전부 |
| uploaded_by | text | 업로더 서명 (`/hansy` 입력 시 `hansy`, 기본 `unknown`) |
| uploaded_at | timestamptz | 자동 기록 |
| cost_krw·impressions·clicks·installs·active_users | numeric **generated** | payload에서 자동 파생(콘솔 조회용, 쓰기 불필요) |

- **UNIQUE(channel, week_start, ad_name)** — 같은 매체·주차·소재는 한 번만. 재업로드 중복은 DB 레벨에서 무시된다.
- **payload가 JSONB인 이유**: 매체마다 지표 스키마가 다르고(메타 ~14종, 몰로코 ~20종, 구글 ~15종) 새 지표가 계속 생긴다. 컬럼 분리 시 대부분 NULL인 25+ 컬럼과 지표 추가 때마다 DDL 변경이 필요해진다. 사이트는 행 전체를 통째로 읽어 브라우저에서 집계하므로 컬럼 단위 쿼리 이점도 없다. 식별·필터 키(channel/week_start/ad_name/업로더)는 정식 컬럼으로 분리되어 있고, 자주 보는 수치는 generated column으로 노출한다 — "쿼리는 컬럼, 내용물은 JSONB" 하이브리드.

payload 필드(합집합): `비용, 노출 수, 클릭 수, 어트리뷰션 수, 활성 유저 수, 신규+재활성 유저 수, _m_installs, _vcrNum, _vcrDen(완주율 분자/분모), _v25~_v100(동영상 25/50/75/95/100% 구간별 재생수 — 있으면), _g_interactions, _g_trueViews, _g_conversions, _g_allConv, _g_convValue` + **유저 질(AF 코호트)** `_q_retD1Base/_q_retD1Users, _q_retD7Base/_q_retD7Users(D1·D7 잔존 코호트 분모/분자), _q_revD0/_q_revD7(코호트 매출 KRW), _q_afInstalls(AF 신규 설치), _q_reAtt(재어트리뷰션), _q_reEng(재참여)` + 문자열 `_m_url, _m_ctype, _m_res, _m_group, _q_campaign, _g_assetStatus, _g_assetType, _g_status, _g_level, _g_url, _g_assetId, _g_adGroupId, _g_campaignId`

### 유저 질(AF 코호트) 데이터 — 주간 통합 소재 리포트
- **출처**: 수퍼셋 크리에이티브 대시보드 dataset 내보내기(`week_start,channel,creative_name,…,cost_krw,…,ret_d1_*,cohort_revenue_*,af_installs,re_engagements,video_play_*`). 몰로코(`moloco_int`)+메타(`Facebook Ads`)가 한 파일.
- **자동 반영**: 사이트가 이 형식을 자동 감지(`isQualityExport`)해 채널별로 분리 파싱(`parseQualityExport`) → 누적·저장. 관리자가 파일만 떨어뜨리면 됨. ₩1,500 미만 행 제외, 파생지표(잔존율=Users/Base, ROAS D7=revD7/비용)는 로드 시 재계산.
- **활용**: 표 '질' 컬럼 그룹(D1/D7 잔존·ROAS D7·신규 설치(단가)·재참여(단가)) · 효율지도 축 · 상세 '유저 질' 탭(잔존 vs 중앙값·주차 추이·코호트 매출·유입 구성·표본 신뢰) · **채점: D1 잔존율 — 몰로코 6%·메타 22%**(SPEC_EVIDENCE den=retD1Base·n0=150, 데이터 없는 매체는 가용성 게이트로 자동 제외) · 자동 해석 '유저 질 최고' 라인.
- 2026-07-10 일괄 마이그레이션: 몰로코 11,357 + 메타 863행 업서트(주간 최신 2026-06-29 포함).
- **2026-07-11 스키마 정리**: CR 테이블 전부 `sr_` 접두사로 통일(`weekly_creative_stats`→`sr_weekly_creative_stats` 등). 프론트는 RPC(함수)로만 접근하므로 함수 이름/시그니처는 그대로 → **사이트 무변경**. RLS 꺼진 채 공개 노출되던 7/10 백업 테이블 2개 + CR 무관 빈 테이블 3개(site_data·persona_site_state·creative_rule_snapshots) 삭제. `creative_names`(19행)는 CR 미사용이나 타 앱 소유 가능성으로 보존.

### sr_scoring_v2 — 정제 채점 뷰 (2026-07 개편 · 분석/BI 전용)
- `supabase/migrations/20260714155424_creative_reporter_scoring_v2_view.sql`(+`155525` 잠금)로 반영된 **읽기 전용 뷰**. 과집계(홈방문·전환) 제외, `nr_users`(AUU)·`ecpa_nr`(신규+재활성 eCPA)·`cpm`·`ctr_pct`·`ret_d1/d7_pct`·그룹 키(`g_adgroup_id` 등)를 노출 — 프론트 채점 정의와 동일한 서버측 원장(교차검증·슈퍼셋용).
- **프론트는 이 뷰를 직접 조회하지 않는다**(기존 RPC 유지 — 03 결정 로그 D7). `security_invoker=true` + `anon/authenticated` revoke로 비밀번호 게이트 우회 차단(Supabase 기본권한이 신규 객체에 SELECT를 자동 부여하는 것 방지).

### sr_kv — 보조 저장소 (k, v jsonb, updated_at, updated_by)
| 키 | 내용 | 채워지는 경로 |
|---|---|---|
| `gmap` | `{adgroup:{id:이름}, campaign:{id:이름}, asset:{항목ID:공식이름}}` | 이름 매핑 CSV·광고그룹 보고서 드롭, 이미지 매칭 배치 |
| `ginfo` | `{광고그룹ID:{nm,cpNm,st,op,rsn,tcpa,cost,…,v25~v100,from,to}}` | 광고그룹 보고서 드롭 |

### sr_users — 사용자 계정과 승인 워크플로
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | bigint identity | PK |
| daangn_name | text | 사용자명(영문 시작 2~24자 영문·숫자). 앞으로 수정·제작·즐겨찾기 등 행위의 서명(수정자) 값 |
| name_lower | text **generated** unique | `lower(daangn_name)` — 대소문자 무관 유일성(사칭 방지) |
| pw_hash | text **unique** | 비밀번호 sha256 해시. 로그인·게이트가 비번만으로 사용자를 식별하므로 전역 유일 |
| pw_enc | bytea | 비밀번호 **복호화 가능 암호문**(pgcrypto pgp_sym, 키는 서버 함수 본문에만). 관리자가 비번을 열람·관리하기 위한 용도 — 관리자 게이트 RPC 안에서만 복호화 |
| status | text | `신청함`(가입 신청, 로그인 불가) · `승인됨`(로그인·데이터 접근 가능) · `거절됨`(로그인 불가, 기록 보존·재승인 가능) |
| **role** | text | `viewer`(보기 전용, 기본값) · `editor`(저장·수정 가능) · `admin`(에디터 상위 — 소재 이름 변경까지). 관리자가 admin.html에서 지정. 현재 admin은 hansy만 |
| created_at·reviewed_at·reviewed_by | | 신청 시각 · 승인/거절/권한 처리 시각·처리자 |

- **비밀번호만으로 로그인**: 입력 비번의 sha256 해시로 사용자를 찾아 이름·상태·권한을 반환(`cr_login`). 그래서 pw_hash를 전역 유일로 강제(가입 시 중복 비번 거부).
- **레거시 `0715`는 이름 없는 뷰어 하드코딩 로그인** — cr_login이 `{viewer:true}`를 반환(이름 없음), 데이터는 읽되 저장·수정은 불가. sr_users 행이 아니므로 가입 시 예약 비번 `0715`는 거부.
- 권한 모델: **뷰어**(0715 또는 role=viewer 사용자) = 읽기 전용 · **에디터**(승인됨 + role=editor) = 저장·수정 가능 · **관리자**(승인됨 + role=admin) = 에디터 권한 + 소재 이름 수동 변경(`/namechange`). 관리자는 에디터 권한을 포함(canEdit=true). 소유자는 계정을 만들어 admin.html에서 자신을 에디터/관리자로 승인. 관리자용 로그인 계정 `admin`(editor) 시드. **관리자 권한(admin role)은 admin.html 마스터 게이트(별도 시크릿)와 다른 개념** — role=admin은 앱 내 최상위 사용자 등급, `_cr_check_admin`은 계정 관리 콘솔용 마스터 비번.

### sr_user_marks — 즐겨찾기 / 내 소재 (개별 계정 단위)
| 컬럼 | 설명 |
|---|---|
| user_name | 마크 소유자 daangn_name |
| channel · ad_name | 대상 소재 (매체, 소재 식별자) |
| kind | `fav`(즐겨찾기, 비공개) · `mine`(내 소재, 등록자 공개) |
- UNIQUE(user_name, channel, ad_name, kind). RLS deny-all — RPC로만.
- **뷰어도 마크 가능**(계정만 있으면). 0715는 계정이 없어 불가. 쓰기 게이트가 아니라 `_cr_uname(pw)`로 비번→사용자 도출(0715/미존재는 null→거부).
- `cr_marks_load`는 **내 소재는 전원 공개(등록자 이름 포함)**, **즐겨찾기는 본인 것만** 반환(프라이버시).

## 3. 보안 모델

- 세 테이블 모두 **RLS deny-all**: anon 키로는 SELECT/INSERT 자체가 불가능.
- 모든 데이터 접근은 **SECURITY DEFINER RPC**를 통해서만 하고, 각 RPC 첫 줄에서 `_cr_check_pw(pw)`가 비밀번호를 검사(불일치 시 예외).
- **읽기 게이트 `_cr_check_pw`**: 입력 비번이 레거시 `0715` 이거나 **승인됨 사용자의 비번**이면 통과(읽기). 신청함·거절됨은 차단.
- **데이터 저장 게이트 `_cr_check_admin_role`** (2026-07-11): `cr_save`(주간 성과)·`cr_kv_merge`의 드롭 유래 키(gmap·ginfo·ocr 등)는 **승인됨 + role=admin(현재 hansy)** 만 통과. 에디터·뷰어의 드롭은 화면 분석만 되고 저장은 서버가 거부(프론트 `isAdmin` 게이트 + 안내 토스트). 예외: `ytt`(유튜브 제목 자동 캐시)만 에디터 허용(`_cr_check_editor` — admin 포함하도록 정상화).
- 클라이언트의 publishable key는 공개되어도 되는 값(설계상 공개키). 실질 게이트는 비밀번호.
- 비밀번호는 브라우저 sessionStorage(탭 단위)에만 저장(`crpw`) — 새로고침 유지, 탭 종료 시 소멸. 로그인한 daangn name은 `crname`(탭)·`cr_uploader`(localStorage, 기기)에 저장돼 업로드·수정 서명에 쓰인다.
- **관리자 게이트**는 사용자 비번과 완전히 분리된 별도 시크릿(`_cr_check_admin`). admin.html에서 런타임 입력만 하고 **리포지토리엔 저장하지 않는다**(GitHub Pages 공개 파일). 관리 RPC(`cr_admin_*`)만 이 시크릿으로 게이트.
- 회원가입·로그인은 anon 키로 공개 호출 가능(신청은 누구나, 로그인은 승인·비번 필요) — 기존 보안 태세와 동일.

### 로그인·가입 흐름
1. **가입**: 랜딩 '회원가입' → daangn name(영문 시작 2~24자)·비번·비번확인 → `cr_register`(신청함 적재). 이름/비번 중복·형식 오류는 서버가 거부(`name_taken`/`pw_taken`/`name_invalid`/`pw_weak`). 성공 시 "관리자 확인 후 가입이 승인됩니다".
2. **관리자 처리**: admin.html에서 신청함 → 승인됨/거절됨. 거절은 삭제가 아니라 상태 저장이라 언제든 재승인 가능.
3. **로그인**: 비밀번호만 입력 → `cr_login`이 `{found,name,status,role,viewer}` 반환. 인사(탭당 1회) — 0715 뷰어="뷰어 권한으로 로그인했어요 👀"(이름 없음) · 승인된 사용자(에디터·뷰어 공통)=**시간대별 랜덤 인사** "환영해요 NAME!"(`greetingFor`) · 신청함="아직 관리자 승인 대기 중이에요" · 거절됨="가입이 거절된 계정이에요" · 미존재=흔들기. (모든 토스트는 당근 보이스 해요체)
   - **로그인 시엔 인사 토스트만**: 첫 로그인(탭당 1회, `willGreet`)이면 `cloudUnlock`의 "☁️ 저장소를 열었어요…" 로드 토스트를 생략하고 환영 인사만 띄운다(예전엔 로드 토스트+환영 두 개가 겹쳐 떴음). 새로고침 자동 재접속(`silent`, 이미 인사함)에선 로드 토스트가 그대로 뜬다.
   - **시간대별 인사(`GREETINGS`/`greetingFor`)**: 분 단위 경계 — 새벽 00–05 · 이른아침 05–09 · **출근 09:00–11:30** · **오전근무 11:30–12:30** · **점심직전 12:30–13:00** · **점심 13:00–14:30** · **오후근무 14:30–18:00** · 저녁 18–22 · 밤 22–24. 각 구간에 3~4개 문구를 두고 랜덤 선택. 브라우저 로컬 시각(`new Date`) 기준.
- **비밀번호 규칙**(`_cr_pwok`): **4자 이상**(복잡도 요구 없음). **같은 비밀번호로 다른 계정 생성 금지**(pw_hash 전역 유일 → register·admin_create·setpw 모두 `pw_taken`로 차단 → "다른 걸로 정해 주세요"). 예약비번 `0715`만 별도 거부.
- **새 가입 신청 알림**(관리자 전용): admin.html이 열려 있는 동안 15초 폴링해 새 신청 시 토스트+브라우저 알림, **거절은 확인창 없이 즉시**. (메인 화면에는 알림 없음 — 관리자만 봐야 하므로)
- **관리자 페이지**: **한 줄 행 표**(컬럼명 '이름'), 비밀번호 열은 평소 ●●● **호버 시 표시**(클릭 복사), 이름 **✎수정**(cr_admin_set p_name). 권한은 **뷰어/에디터 2토글**(승인 기본 viewer) — role=admin은 콘솔에서 부여·강등 불가, 주황 `관리자` 태그로만 표시. **대기 버튼 없음**(승인·거절 안 누른 신청함=대기). **신청일·처리는 '상세 보기' 체크 시에만** 표시. 반응형 열 숨김: ≤860 비밀번호 → ≤660 비번변경·✎ → ≤520 권한(이름+승인/거절 최우선 유지). ＋계정 추가·비번변경.
- **즐겨찾기 / 내 소재**(로그인 계정, 뷰어 포함 · 0715 제외): 소재 이름 우측 별 버튼 2종 — ★ 즐겨찾기(켜면 노랑) · 체크 인장 내 소재(켜면 하늘색). **표시 시(on) 별은 호버 없이 상시 노출**, 미표시는 호버 시에만 회색. **즐겨찾기는 별만**(칩·글자 없음) · **내 소재는 남이 등록한 것만 등록자 이름 칩**(본인 것은 켜진 하늘색 별로 충분하므로 칩 생략). 별은 **전체 목록·그룹 행·상세의 "같은 그룹 내 비교" 목록** 모두에 표시. 좌측 nav "보기": 전체 / 내 소재 / 즐겨찾기 필터.
  - 마크는 로그인 직후 비동기(`cr_marks_load`)로 채워진다. 최초 표는 `MARKS.me`가 아직 없는 채로 그려지므로, 마크 로드가 끝나면 **현재 탭과 무관하게 표를 다시 그려**(`loadCloudRows`) 별·칩을 채운다. (예전엔 `mainTab==="table"` 게이트에 막혀 기본 탭 `analysis`에선 로그인 직후 전체목록 별이 안 보이던 버그가 있었음)
- **내 소재 등록 대상(관리자 전용 드롭다운)**: 표 툴바의 🔖 단일선택 드롭다운(`#mineAsMsel`, 상태 `mineAs`)에서 로스터 타인을 고르면, 이후 **인장 별 토글·"내 소재" 보기 필터가 그 사람 기준**으로 동작한다. 남을 고른 상태의 별은 본인용 `cr_mark_set` 대신 **`cr_admin_mark_set`**(특정인 이름의 mine 마크 대신 등록/해제)로 저장되고, 대상이 남이면 인장 별을 **주황(`.as`)**으로 구분한다(등록자 칩으로도 확인). '나 (본인)'로 되돌리면 원래대로. 비관리자에겐 드롭다운이 숨겨지고 `mineAs`는 항상 무시된다. (연필 팝오버가 개별 소재 다인 편집 경로라면, 이 드롭다운은 여러 소재를 그 사람 이름으로 연속 태깅하기 위한 경로)
- **그룹 마크(파생·일괄)**: 그룹뷰에서 그룹 별을 누르면 멤버 전원을 일괄 토글(`cr_mark_set_many`). 그룹 상태는 **저장하지 않고 멤버에서 파생** — 멤버 전원이 표시면 그룹도 표시로 보이고, 멤버 하나를 개별 해제하면 그룹 표시도 자동 해제. 표·비교 목록은 전체 재렌더 없이 제자리 갱신(`refreshMarksUI`)해 열린 상세가 닫히지 않는다.
- **제작자 로스터**: 제작자(내 소재 등록자)는 항상 고정 로스터 **vin·mia·tank·hansy·marin·jace** 에서만 고른다(직접 입력 없음). 연필 팝오버의 제작자는 로스터 토글칩(복수), 필터는 로스터 6 + **기타**(로스터 제작자 없음). (기존 `admin` 등록분은 마이그레이션으로 `hansy`에 이관)
- **소재 편집(관리자, 연필 팝오버)**: 별(즐겨찾기·내 소재) 오른쪽 **연필 버튼**(관리자·개별 소재만, 호버 노출) → 그 소재의 **이름·서비스 분류·제작자** 를 한 팝오버에서 수정. 저장 시 이름=`cr_names_merge`(kv `nameovr`), 서비스=`cr_svc_merge`(kv `svcovr`, ''=자동 복원), 제작자 로스터 토글 diff=`cr_admin_mark_set`(특정인 mine 마크 대신 등록/해제). 표시 이름 우선순위 최상위, 서비스 오버라이드는 `svcOf`·드롭다운·서비스 칩(주황)에 즉시 반영.
- **소재 이름 일괄 변경(관리자, `/namechange`)**: 검색창에 `/namechange`+엔터 → 모든 소재명 contenteditable, 다시 `/namechange`로 저장(`cr_names_merge`). 저장 전 이탈은 `beforeunload`로 붙잡음. (연필 팝오버가 개별 편집의 주 경로, `/namechange`는 일괄용)
- **서비스·제작자 복수선택 드롭다운 필터**: 툴바에 `전체 서비스`·`전체 제작자` 2종(체크박스 복수선택, 팝업 유지). 서비스=키워드 6버킷+기타(관리자 오버라이드 우선). 제작자=로스터 6+기타(항상 노출). 개수는 현재 화면(개별/그룹) 기준.
- **자료 추출(사이드바 `내 자료 추출하기`)**: 중앙 플로팅 모달 — **제작자**(모두·로스터 단일) · **매체**(가로 세그먼트, 순서 모두·몰로코·구글·메타, 복수·모두=전체) · **기간**(가로 세그먼트 1달·3달·**올해**(당해 1월~)·전체·**직접 설정=정확한 시작~종료일 date picker**) · **비교 데이터 3단**(없음/요약만/전체 표 — 기본 **요약만**: B 섹션에 비교군 기준선 한 줄 + 지표별 중앙값·25/75백분위 통계표 + 상위 3개 참고만 담아 전체 표 대비 텍스트를 대폭 절감, 제작자=모두면 없음 강제) · **최소 점수**(없음/40+/50+/60+/70+ — 미만·무점수는 표에서만 제외하고 **요약 통계는 전체 기준 유지 + 제외 수 명시**로 체리피킹·AI 혼동 방지). 순위는 `#r/N`(그 매체·기간 채점 N개 중 r등). 모달·버튼은 아이콘 없는 미니멀, 제작자는 4열 그리드(모두·vin·mia·tank / hansy·marin·jace). 매체별로 그 기간에 맞춰 **재집계·재채점**(dataset 복제 후 `scope()`, 절대 구간은 `scopeRange()`)한 소재 성과를 **AI가 읽기 좋은 Markdown**으로 내려받음. 매체마다 A(대상 제작자)·B(동조건 타 제작자 비교군, 체크 시) 섹션으로 분리하고 각 소재는 자기 행에만 지표를 담아 **절대 혼용되지 않게** 한다. 점수는 매체·기간 비교군 상대치라 매체 간 직접 비교 금지를 헤더에 명시. (CSV 넣기 버튼은 아이콘으로 축소 + 드래그드롭 유지)
- **OCR 마크**: 소재명 옆이 아니라 **상세보기 미리보기의 애셋 URL 왼쪽**에 표시. **Live**는 소재명 왼쪽 **초록 점**(글자 없음).

## 4. 파일 인식과 저장 파이프라인

드롭된 파일은 이 순서로 분류된다:

1. **구글 확장 소재 연결 보고서** — 헤더에 `애셋 상태`+`확장 소재`. `주` 컬럼(주간) 파싱, 지표 매핑(비용/노출수/클릭수/설치/인앱 액션/전환 등 → 내부 키).
2. **몰로코 크리에이티브 리포트** — `creative_preview` 또는 `ad_name`+`creative_type`. 미리보기 CDN 링크·타입·해상도·그룹 추출. **네이티브 영상 오등록 보정**: 리포트가 완주율을 제공하는데 `creative_type`이 VIDEO인 소재의 첨부가 이미지 파일이고 완주율이 전 기간 비어 있으면 실제 노출물을 이미지로 보고 이미지로 재분류(칩 `이미지*`·유형 '이미지(영상 등록)'로 표기). 완주율이 찍혀 있으면 미리보기만 썸네일인 진짜 영상이므로 유지, 완주율 컬럼 자체가 없으면 판단 보류.
3. **일반 성과 CSV(메타 등)** — `date`+`ad_name`(또는 세트/그룹 레벨 컬럼). `thumbnail_url` 류 컬럼이 있으면 미리보기 자동 연동. `동영상 … 25/50/75/95/100%` 재생수 컬럼이 있으면 자동 인식해 `_v25~_v100`으로 누적 — 상세 '영상 시청' 탭의 구간별 유지 곡선 데이터(율·평균·시간·비용류 헤더는 제외, 횟수만).
4. **광고그룹 보고서** — 헤더에 `광고그룹 상태`+`광고그룹 ID`. 그룹 메타+누적 성과+영상 재생 퍼널을 `ginfo`로, 이름을 `gmap`으로.
5. **이름 매핑 CSV** — `광고그룹 ID`+`광고그룹` / `캠페인 ID`+`캠페인` / `항목 ID`+`애셋` 쌍. 제목 줄이 앞에 있는 UI 내보내기도 지원.

**매체 판별 우선순위**: ① 리포트 형식 자동 감지 → ② 미리보기 URL 도메인(adsmoloco→Moloco, googlesyndication/youtube→Google, fbcdn→Meta) → ③ 드롭 위치 → ④ 파일명 접두(meta_/moloco_/google_).

**저장 규칙**:

`displayNameOf(entity)` 우선순위:

4. **유튜브 제목** — `ytt[영상ID]` (로드 직후 전 소재 자동 조회·kv 공유)
5. 원문 (data-name·저장·병합 키는 항상 원문 유지, hover 시 원문 툴팁)

### 오프라인 이름 보강 배치 (필요 시 재실행)
- **비전 OCR(현행 방식)**: 이름이 `tpc.googlesyndication.com/simgad/…` URL로 남은 구글 이미지 애셋을 사람이 읽을 헤드라인으로 채운다. **사이트 접속과 무관한 오프라인 사전 채움**이 원칙 — 브라우저에서 즉석 OCR하지 않고 `ocr` kv에 미리 넣어둔다.
  - 절차: ①gap 조회(아래 SQL) → ②이미지 다운로드 → ③각 이미지를 비전으로 읽어 **가장 큰 헤드라인 한 줄만 한국어 verbatim** 추출(로고 워드마크·CTA·법적고지 제외) → ④`ocr` kv에 `{항목ID:헤드라인}` jsonb 병합(`v = v || '{…}'::jsonb`). 이 이미지들은 이미 Google 공개 광고 CDN에 게시된 라이브 크리에이티브라 노출 민감도가 낮다.
  - gap 조회: `sr_weekly_creative_stats`의 `_g_assetType`이 이미지인 distinct `_g_assetId` 중 `ocr`·`gmap.asset` 어디에도 없는 것 = OCR 필요분. **마이그레이션 직후 항상 이 조회로 0인지 확인**하고 남으면 위 절차를 돌린다.
  - 품질 주의: 크롭으로 가격/숫자가 잘린 소재는 잘못된 값(예 `0원`)을 단정하지 말고 일반 문구로. `삶은당근` 같은 브랜드 카피는 그대로 유지.
- **PaddleOCR(korean, 과거 방식·참고)**: 200장 일괄 OCR. `OMP_THREAD_LIMIT=1`, 숫자 라인 신뢰도 0.55, 박스 좌표 기하 정렬, 로고 오인 후처리. 전문(全文) 덤프라 이름이 길고 지저분 → 신규분은 비전 방식으로 대체.
- **dHash 이미지 매칭**: 몰로코 IMAGE 소재(이름 있음)와 구글 simgad 이미지의 지각 해시(9×8 dHash, 해밍 ≤6 + 종횡비 ±12%)로 동일 이미지를 찾아 몰로코의 실명을 구글 애셋에 부여 → `gmap.asset`.
- 취소: 해당 kv 키 삭제 한 번이면 전체 원복.

ROAS·활성 eCPA 등 검증이 약한 지표는 라이브러리에서 **'고급 · 채점 비권장'** 그룹(점선 구분·회갈색 라벨)으로 분리해 여전히 추가는 가능하되 기본엔 안 든다(`MET_META[...].disc:true`). **저장 가드**: 어떤 매체든 효율 지표(비규모)가 전부 0/제거면 규모만으론 채점할 수 없어 저장을 막고(모달 하단 `#scErr` 인라인 경고 + 문제 매체 탭으로 이동), 편집하면 경고가 사라진다.

- **사이드바(다크) — 간결·중요도 우선**: ① **화면 전환 탭**(분석/전체 목록/AB) 최상단 — 본문 헤더에서 좌측 nav로 이동(활성=주황) → ② 필터(**매체=커스텀 드롭다운**(네이티브 select 대체, 팝업 메뉴 — 각 항목에 **데이터 보유 범위**(첫 주~마지막 주·주 수, `chDataRange`) 표기, 버튼 툴팁 동일) · **기간**=1달/3달/전체 세그먼트+아래 '직접 설정'(최근 N 주/달 · **월별 보기**(월 승자+월내 추이, 주 시작일 기준) · **날짜 범위** from~to — P6, 2026-07 회의)+아래 **💾 보유 데이터 범위 줄**(현재 매체, 기간 필터와 무관) · **보기**=전체/My/★ 세그먼트) → ③ **액션 행**(⚙ 채점 설정 · 자료 추출(주황) · 용어 ⓘ). 요약 KPI 패널·CSV 넣기 버튼·☁️ 저장소 줄 제거(CSV는 드래그&드롭 업로드, 새로고침은 페이지 리로드). **모바일(≤900px)**: 매체·기간·보기를 가로 3열 드롭다운으로 압축(직접 설정 입력은 필터 아래 전폭), 액션 3버튼 한 줄. 데스크톱(좁은 컬럼)은 채점 설정 한 줄+자료 추출·용어 다음 줄.
- **채점 설정 지표 카탈로그**: 집계 시 실제 파생되는 지표만 노출 — 단가류(신규+재활성/활성 eCPA·eCPI·신규 설치/재참여 단가·신규/홈 단가·CPA·CPM·CPC), 반응(CTR·완주율·신규 전환율/전환율·상호작용율·조회율·홈 도달율), 질(D1/D7 잔존·ROAS D7 AF실측·ROAS 모델). 각 지표는 **↑/↓ 방향 대신 작은 회색 한 줄 설명**(desc)만 표시(좋고 나쁨은 표·지도가 표현). MET_META에 desc, SPEC_EVIDENCE에 신규 지표 표본 축소 추가.
- **전체목록 상단 스티키 툴바(2줄)**: 툴바(`.tbar`)는 `position:sticky;top:0`로 스크롤해도 상단 고정 — 1줄=검색 · [서비스·제작자 복수선택] · 칩필터 · [라이브·비디오·이미지 세그먼트] · [그룹뷰·갤러리뷰·비활성 포함 세그먼트]. 표 헤더(`thead th`)도 `position:sticky`지만 `top:var(--tbarH)`로 툴바 높이만큼 내려 붙어 2번째 스티키 줄이 된다. `--tbarH`는 `syncTbarH()`가 `#tabTable`에 툴바 실측 높이를 심어 캐스케이드시키며, `renderQuickFo()`·resize에서 갱신. 갤러리뷰·비활성 포함은 세그먼트 버튼(체크박스 아님)으로 토글.
- **'선택됨' 표시 체계(표면별 1가지)**: 강조가 화면마다 제각각이던 것을 정리 — **탭=주황**(사이드바 네비 탭 + 채점 모달 매체 탭), **활성 필터=진회색(ink)**(서비스·제작자 드롭다운 활성·칩필터·라이브/홈도달 토글·필터 팝업 칩), **세그먼트 토글=중립 흰 알약**(비디오/이미지·그룹뷰 등). 주황은 네비/브랜드에만 쓰여 위계가 선다.
- **전역 오류 처리**: `logErr(ctx,e)`로 콘솔 집계, `window.onerror`/`unhandledrejection`에서 ResizeObserver·크로스오리진 잡음은 무시하고 그 외엔 토스트로 노출. CSV 업로드는 파일 하나 파싱 실패가 전체 배치를 막지 않도록 개별 `try/catch/finally`로 격리(실패 파일명만 수집).
- **세그먼트 탭**: 분석 / 전체 목록 / AB테스트(인식 그룹 0개면 탭 숨김). 본문은 카드 없이 흰 배경 풀폭(최대폭 1240px 중앙 정렬), 푸터는 캔버스색+간격으로 구분.
- **분석**: 요약 타일(포맷/사이즈 우위, 주간 흐름) · 효율 지도(2축 산점도 — '보기' 세그먼트로 개별 소재 또는 포맷/사이즈/형태/지면/서비스별 집계(상하 10% 트림) 전환, 가로/세로 축도 세그먼트, 범위 슬라이더. 구 '유형별 비교' 그래프는 지도에 흡수) · 주차별 추세(선택 기간 전체, 라벨 자동 솎기).
  - **효율 지도 컨트롤 바(한 줄)**: `필터`(**Live 단독 토글 버튼** + **Video/Image 세그먼트**) · **보기·가로·세로 = 드롭다운(select)** 을 가로로 나란히(`.qbar`). 보기는 **소재/사이즈/서비스** 3종만(값 2종 이상 차원만 노출). 기본 축 = 매체 채점 스펙 유도(defaultAxes): 세로 채점 1순위 단가 × 가로 성격이 다른 무조건부 상승 지표 — AEO 신규 단가×신규 전환율 · ACE 홈 단가×홈 도달율 · 몰로코/메타 eCPA×D1 잔존율 (스펙 지표 없으면 폴백 체인). Live/Video/Image는 지도에 그릴 소재 집합에만 적용(표 필터와 독립), 데이터셋 전환 시 리셋. 개별 소재 보기는 **원 위 이름 라벨 생략**(겹침 방지·호버로 확인), 유형 보기만 라벨. 원 겹침은 호버 강조(불투명도·외곽선↑)+양축 슬라이더로 좁혀 봄.
  - **주차별 추세 주 지표 토글**: 광고비/노출/클릭/설치/활성유저 중 선택(데이터 있는 지표만 노출) — 유저 반응 절대량 흐름 확인용. 효율선(단가)은 광고비 선택 시에만 함께 표시.
- **전체 목록**: 소재명 **왼쪽**에 라이브(`.lvb` — **초록 점**만, 글자 없음)·학습중(`.lrn`) 뱃지, 소재명 우측 `›` 캐럿은 제거(행 전체가 펼침 토글). 컬럼 계층(대표: 점수·**광고비**·**데려온 유저**(ACE는 '데려온 방문'=홈)·홈 방문(AEO)·채점 1순위 단가·D1 잔존(몰·메타)·CTR·완주율(몰 영상)·ROAS(참고), 헤더 +로 세부 펼침) · 창 좁으면 중요도 낮은 그룹부터 자동 숨김(fitTableCols: 참고→반응→영상→유저 질→홈→광고비 · 데려온·단가·점수 불가침 · 표 위 안내) · 기본 Live 필터(스코프 최신 집행 주 광고비>0) + 무점수(–) 소재 기본 숨김('비활성 포함'으로 표시) · GroupView(광고그룹 집계, Video/Image와 상호 배타) · 이름 아래 칩 클릭=필터(서비스·포맷·사이즈·그룹·날짜) · **제작자 칩** 클릭=그 제작자 필터 토글(‘전체 제작자’ 드롭다운과 동일). **칩은 그룹명 칩(보라)만 색을 쓰고 나머지(서비스·포맷·사이즈·제작자·날짜)는 전부 무채색(회색)** — 선택된 제작자 칩만 진회색 반전으로 활성 표시 · 60행 페이지. 점수 칸은 10단계 색(십의 자리마다 다른 색, 파랑→청록→초록→올리브→무채→황토→주황→빨강).
- **미리보기 실측 사이즈·재생시간(MDIM)**: 이름/리포트에 해상도 표기가 없는 매체(메타·구글 등)도 **미리보기에서 크기를 얻는다**. 이미지는 이미 뜨는 썸네일 `onload`의 `naturalWidth`, 영상은 상세 재생 `loadedmetadata`(+ 접속 2.5초 뒤 시작하는 스로틀 백그라운드 `preload=metadata`, 동시 2·상한 240)로 실측해 **기기 `localStorage`(`cr_mdim`)에 캐시**(서버 무접촉). 값은 디코드된 원본에서 읽어 정확하고, 프록시(wsrv.nl 리사이즈본)로 뜬 이미지는 오측 방지로 건너뜀. 새 치수가 들어오면 크기 없는 소재의 `tags.size`/`shape`를 채워(재집계 없이 현재 뷰 반영) 표 사이즈 칩·분석 '보기 → 사이즈'가 **모든 매체**에서 동작. 재생시간은 상세 미리보기에 `1080×1920 · ⏱ 0:15` 형태로 표기.
- **이름 옆 뱃지**: `Live`(초록 점 — 최신 집행 주에 광고비가 나간 소재/그룹, 전 매체 공통) · `경고`(호박색 — 구글 광고그룹 보고서상 정책 제한, 사유는 툴팁) · `학습중`(집행 2주 미만 또는 전환 30건 미만). 일시중지는 뱃지 없음 — Live 뱃지 부재로 드러나고, 그룹 상세 메타에만 '일시중지' 텍스트 표기.
- **상세(행 클릭)**: 한 줄 3열 [고정 미디어 프레임 | 그래프 탭 | 같은 그룹/시리즈 비교] — 3분할 구조 고정. 가운데 열은 세그먼트 탭 3종, 있는 데이터의 탭만 노출(1개면 탭 줄 숨김)하고 선택 탭은 소재 이동(←/→·비교 클릭) 간 유지: ① **주차별 추이**(막대 광고비+지표선, 매 막대 아래 MM/DD와 **그 주 점수** — 그 주 데이터만으로 재채점한 상대 점수(weeklyScoresOf, 스코프 캐시), 티어 색·부적격 주는 –) ② **영상 시청**(영상 소재만) — 구간별 시청 유지 곡선(노출=100 기준, 최대 이탈 구간 음영·−%p 라벨. 데이터 우선순위: 소재 자체 `_v25~_v100`÷노출 → 구글 광고그룹 보고서 누적 퍼널(그룹 단위임을 캡션 명시) → 구간 데이터 없는 매체는 완주율 큰 값+코호트 중앙값 대비) + 완주율/TrueView 조회율 주차별 추이(영상 중앙값 점선) ③ **포지셔닝** — 두 축 모두 '계정 내 위치(광고비 가중 백분위, 방향 보정 0~100)'인 사분면 산점도: 어떤 지표를 골라도 오른쪽·위=좋음. 방향 표시는 미니멀하게 — 중앙값 십자선에 나쁨(빨강)→좋음(초록) 그라데이션 + 선 끝 +/− 글리프만(사분면 틴트·코너 문구는 점과 겹쳐 제거). 축 지표는 가로/세로 토글로 교체(기본 = 효율지도와 같은 defaultAxes 규칙: 세로 채점 1순위 × 가로 성격 다른 주요 지표. 전환력 대표 축은 뷰스루 혼입 전환율 대신 신규 전환율. 선택은 소재 이동 간 유지). 회색=타 소재·파랑=같은 그룹·주황=이 소재, 점 크기 광고비, 점 클릭=해당 소재로 이동. 원값 스케일 분포는 분석 탭 효율 지도가 담당(역할 분담). + 지표별 계정 내 위치 막대. 점수·진단 블록 없음(점수는 행에서, 산정 방식은 푸터에서). 비교 목록은 한 줄 말줄임+공통 접두 '…' 접기(풀네임 툴팁). 프레임은 비율별 3종(세로 232×396/정방 280×280/가로 400×250), 모바일(≤860)은 미디어→비교→그래프 순 1열 스택+프레임 폭 캡. 그룹 상세는 미디어 자리에 보고서 추가 정보(동영상 재생 퍼널은 '영상 시청' 탭이 담당). 키보드 ←/→·Esc, 딥링크 해시 기록.
- **이미지 로딩 복구**: 광고차단 DNS가 광고 CDN(googlesyndication·adsmoloco)을 거부하는 환경 대응. 실패 즉시(대기 0ms) wsrv.nl 프록시로 전환 → (1.2s) 원본+캐시우회 → (1.2s) 프록시+캐시우회 → 실패 표시. 프록시로 성공하면 `crImgProxy`를 localStorage에 학습해 그 기기는 재방문 때도 처음부터 프록시로 렌더(우회 재시도 자체가 없어짐). 모든 이미지 태그는 `data-osrc`에 원본 URL 보존. 호버 툴팁의 유튜브 썸네일은 표에서 이미 캐시된 저해상도(default.jpg)를 즉시 띄우고 고해상도(hqdefault)가 준비되면 교체.
- **AB테스트**: 이름 접미 변형 자동 인식(`_A`/`_B`, v1·v2, 제로패딩 01~05, ep·편·화 회차, 문자 직결 숫자 1~5 — 오인 방지 규칙 다수). 목록(이름+🥇🥈🥉)→펼침 비교표(유의미 차이만 색상, 5% 스프레드 기준)+대표 미디어.
