# CreativeReporter 채점 가이드 (팀 배포용 원고)

> 근거 문서: Obsidian 「CreativeReporter_개편_202607」(00 회의록 · 02 BigQuery 교차검증 · 03 결정 로그 D1~D5) · 구현: `index.html` `SCORING`/`runScoring` · 서버 원장: Supabase `sr_scoring_v2` 뷰.

## 한 문장 요약

**부풀려진 매체 지표(홈방문·전환 CPA)를 걷어내고, '신규+재활성 유저(AUU)'를 기준으로 — 매체 안에서(구글은 애드그룹 안에서) — 지표별 등수를 매겨 효율 50 : 규모 50으로 합산한다.**

## 1. 점수가 계산되는 4단계

1. **매체별 핵심 지표 2개를 잰다** — 효율(단가) 1개 + 규모 1개 (아래 표).
2. **지표마다 '전체 중 몇 등'으로 바꾼다** — 나누기 파생식이 아니라 등수(소재 1개 = 1표). 절대량이 아니라 상대 위치라, 노출 절대량이 적은 이미지 소재가 구조적으로 불이익받지 않는다. 비디오·이미지는 분리하지 않고 함께 순위.
3. **표본·지출이 적으면 50 쪽으로 당긴다**(shrinkage) — 소액 소재의 운빨 최상·최하위 차단. 광고비 하위 15%는 아예 채점 제외.
4. **효율 50 : 규모 50으로 합친다** — 표시 점수는 비교군 내 위치로 재배치(1등 90점대 · 평균 50 근처).

## 2. 매체별 채점 기준 (회의 확정)

| 매체 | 캠페인 | 효율 (50%) | 규모 (50%) | 비교 범위 |
|---|---|---|---|---|
| 구글 | AEO (앱설치) | 신규 단가 (생신규 eCPA) | 노출량 | **같은 애드그룹 안에서만** |
| 구글 | ACE (리타게팅) | CPM | 노출수 | **같은 애드그룹 안에서만** |
| 몰로코 | 전체 | 신규+재활성 eCPA | 데려온 유저 (AUU) | 매체 전체 (그룹뷰 불필요) |
| 메타 | 전체 | 몰로코와 동일 | 몰로코와 동일 | 매체 전체 |

- **매체 간 점수 비교 금지** — 신규+재활성 eCPA가 매체 간 40~80배 차이(실측: Google ₩30 / Moloco ₩684 / Meta ₩2,365).
- **메타 기준은 Marin 정리본과 정합성 확인 예정.**

## 3. 채점에서 뺀 지표와 이유 (과집계)

| 뺀 지표 | 왜 | 실측 근거 (BigQuery 교차검증) |
|---|---|---|
| 홈 방문(수) | 유저가 다음날·모레 재방문할 때마다 누적 | first_open의 **≈427배** |
| 홈 단가·홈 도달율 | 분모/분자가 위 홈 방문 | 〃 |
| ACE 전환 CPA·전환율 | 인앱 전환 중복 집계 (완전 과집계) | 전환 = first_open의 **≈565배** |
| 구글 '제목(headline)' 성과 | 영상/이미지 기여 분리 불가 (텍스트 애셋은 집계 자체 제외) | 소재합의 first_open 귀속률 36~94% |
| 구글 '매우 좋음' 등급 | 반응 퀄일 뿐 유저 퀄 판단 불가 | 부동산 2509: 전부 90점대인데 유입 적음 |
| 활성 유저 수(규모로) | 신규+재활성의 **≈11.6배** 부풀림 | → 규모는 AUU |
| CTR | eCPA와 비상관 (참고용으로만 표시) | 회의 합의 |

- 전부 **표에는 '참고'로 남아 있다** — 채점에만 안 들어간다. 필요하면 ⚙ 채점 설정 라이브러리에서 수동 추가 가능(비권장 그룹).

## 4. 왜 구글만 그룹뷰인가

- 구글은 BigQuery에 **캠페인·애드그룹까지만** 쌓여 광고(소재) 단위 유저당 단가를 볼 수 없다. 소재 단위는 비용은 잘 귀속(83~99%)되지만 first_open은 36~94%만 잡힌다(제목 클릭 기여 유실).
- 그래서 **① 애드그룹끼리 비교해 '그룹의 승자'를 보고(그룹 행 점수) ② 그 그룹 안에서 소재 등수(소재 점수)를 본다.** 잘나가는 그룹의 2~5등이 1등에게 노출을 뺏겨 가려지는 왜곡, 약한 그룹 1등의 과대평가를 막는다.
- 구글 데이터셋을 열면 그룹뷰가 기본으로 켜진다.

## 5. 해석 원칙 (점수를 읽는 법)

- **광고비 = 머신러닝 신호.** 매체는 잘 나가는 소재에 예산을 몰아준다 → 광고비 상위 + eCPA 양호 = 위너 후보, **광고비 상위인데 eCPA도 나쁘면 OFF 후보**. (분석 탭 '자동 해석'에 자동 표시)
- **반응 좋음 ≠ 좋은 유저 획득.** 점수(효율)가 높아도 규모(AUU)가 작으면 위너 아님 — 그래서 규모가 점수의 절반.
- **매체 지표 단독 신뢰 금지.** 크리테오 등 타 매체가 열리면 기여를 뺏겨 실제는 하락해도 구글 지표는 안 떨어져 보인다(카니발라이제이션). 실수치는 **BigQuery(슈퍼셋) 교차검증** 병행 — 구글 CPA의 신뢰 소스는 BigQuery 애드그룹 CPA(소스 확정: Jace 컨펌 대기).
- **D7·D1 잔존율은 보조 지표**(D7이 주 보조) — 유저 질 확인용. 바로구매 잔존 대시보드(Shayna 협업) 기준과의 연동은 향후 검토.
- **몰로코 사이즈 가이드** — 사이즈별 eCPA가 매체 중앙값의 1.5배 이상(표본 3개↑)이면 '끌 지면·만들지 말 소재' 후보로 자동 해석에 표시(예: 숏폼은 잘되는데 이미지가 5만 원대로 뜨는 경우).
- **eCPA는 '유입 유저당 단가'** — 설치(CPI)가 아니다. 삭제 후 재설치 유저가 CPI에 안 잡히므로 신규+재활성 유입 기준으로 본다.

## 6. 자주 묻는 질문

- **광고비를 늘리면 불리한가?** 아니다. 등수는 소재 1개=1표(광고비 비가중)이고, 소액 쪽 운빨만 표본·지출 축소와 광고비 하위 15% 컷으로 차단한다.
- **점수가 낮은데 그룹에서 1등이면?** 구글 소재 점수는 '그룹 내 등수'다. 그룹 자체가 약하면 그룹 행 점수(그룹 간 비교)가 낮게 나온다 — 둘을 같이 보라.
- **월별 성과는?** 사이드바 기간 → 직접 설정 → '월별'에서 달을 고르면 그 달의 승자(표)와 월내 추이(주차별 추세)를 함께 본다. 날짜 범위(from~to) 직접 입력도 가능(주 시작일 기준).
- **커스텀 채점은?** ⚙ 채점 설정에서 기기별로 지표·비중 조절 가능. 기본값과 같으면 저장되지 않아 팀 표준(위 기준)을 따른다.
